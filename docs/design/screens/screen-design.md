## 3.3 共通UI設計

### レスポンシブ方針

| デバイス | UI方針 |
|---|---|
| PC | 高情報量UI |
| スマホ | カードUI中心 |
| 共通 | 検索バー上部固定 |

---

### 検索UI

#### PC
- 画面上部固定
- インラインフィルタ表示
- 即時検索対応

#### スマホ
- 画面上部固定
- フィルタボタン押下でBottomSheet表示
- 検索条件を縦並び表示

---

### 新規登録UI

#### 基本方針
- メモは必ず書籍に紐づく
- メモ追加は書籍詳細画面から行う
- ホーム画面や検索画面にはメモ追加ボタンを配置しない
- 画面ごとにFAB（スマホ）/ コンテンツエリアボタン（PC）の役割を切り替える

### 共通タグ入力UI仕様

| 項目 | 仕様 |
|---|---|
| 入力方式 | テキスト入力 + サジェスト（既存タグを前方一致で表示） |
| 新規作成 | 既存タグに一致しない文字列を入力してEnterまたは「追加」ボタンで新規タグとして登録 |
| 削除 | タグ表示の × ボタンで当該メモからの紐付けを解除（タグ自体は削除しない） |
| 上限 | 特に設けない |
| 最大文字数 | 50文字以内（§7.3参照） |
| 重複 | 同一タグの二重追加は無視する |

#### UIイメージ

```text
[ #DDD  ×  ] [ #設計  ×  ] [ タグを入力... ]
                              ↓ 入力中サジェスト
                            ┌─────────────┐
                            │ #設計        │
                            │ #設計パターン│
                            └─────────────┘
```

---

### 追加読み込み仕様（SCR-06・SCR-09共通）

| 項目 | 仕様 |
|---|---|
| 初期表示件数 | 50件（created_at DESC） |
| 追加読み込み | 「さらに読み込む」ボタン押下で50件追加取得 |
| ボタン表示条件 | 取得件数が50件未満の場合は非表示（全件表示済み） |
| 検索・フィルタ時 | 条件変更のたびに先頭50件にリセット |
| 実装方式 | URL Query Parameter に `page` または `offset` を持たせず、クライアント state で管理（検索条件はURLパラメータで管理） |

- 画面別のアクションボタン（「＋書籍追加」「＋メモ追加」）はコンテンツエリア上部（検索バー横）にstickyで表示する。

#### スマホ

- ヘッダー左端にアプリ名・ロゴを表示する。
- ヘッダー右端にハンバーガーアイコン（☰）を表示する。
- アクションボタン（「＋書籍追加」「＋メモ追加」）はFABとして画面右下に表示する（ヘッダーには表示しない）。

##### ドロワー仕様

| 項目 | 内容 |
|---|---|
| 表示方向 | 右側からスライドイン |
| 表示内容 | ユーザー情報セクション（上部）＋グローバルナビゲーション4項目＋設定・ログアウト（下部） |
| 閉じ方 | ✕ボタン押下、またはドロワー外領域タップ |
| オーバーレイ | ドロワー展開中は背面をオーバーレイで暗転する |

##### ドロワー構成

| セクション | 内容 |
|---|---|
| ユーザー情報 | ユーザーアイコン＋ユーザー名（タップ不可・表示のみ） |
| ナビゲーション | ホーム・書籍一覧・全メモ検索・お気に入り |
| アカウント操作 | 設定（/settings へ遷移）・ログアウト（`signOut()` 呼び出し後 /login へリダイレクト） |

##### UIイメージ（スマホ・通常時）

```text
┌──────────────────┐
│ memoLake        ☰ │
└──────────────────┘
```

##### UIイメージ（スマホ・ドロワー展開時）

```text
┌──────────┬────────┐
│          │      ✕ │
│          │ 👤     │
│          │ 山田太郎│
│  （暗転）  │ ──────  │
│          │ ホーム  │
│          │ 書籍一覧 │
│          │全メモ検索│
│          │お気に入り│
│          │ ──────  │
│          │ 設定   │
│          │ログアウト│
└──────────┴────────┘
```

---

#### ログインユーザー表示仕様

| 項目 | 内容 |
|---|---|
| 表示値 | `user_metadata.name`（Supabase Authに保存したユーザー名） |
| 取得タイミング | レイアウトコンポーネント（`(protected)/layout.tsx`）でサーバーサイド取得 |
| PC | ヘッダー右端にユーザー名テキスト＋ドロップダウン（設定・ログアウト導線） |
| スマホ | ドロワー上部にユーザーアイコン＋ユーザー名を表示（表示のみ）、ドロワー下部に設定・ログアウト導線 |

| 項目 | 内容 |
|---|---|
| PC ドロップダウン項目 | 設定（/settings へ遷移）、ログアウト |
| ログアウト処理 | `signOut()` Action を呼び出し、完了後 `/login` へリダイレクト |
| PC 閉じ方 | メニュー外領域クリックで閉じる |
