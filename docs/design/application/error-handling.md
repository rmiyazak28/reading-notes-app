# 10. ローディング・エラーUI設計

## 10.1 ローディングUI方針

### 基本方針

| 種別 | 実装方法 | 用途 |
|---|---|---|
| ページ単位 | `loading.tsx`（スケルトンUI） | 初期データ取得中 |
| アクション単位 | ボタンのdisabled + ローディング表示 | 保存・削除などの操作中 |
| お気に入りトグル | 楽観的UI（即時反映） | ★ボタン押下時 |

### スケルトンUI対象画面

| 画面 | スケルトン対象 |
|---|---|
| SCR-03 ホーム | 書籍カード・メモカード |
| SCR-04 書籍一覧 | テーブル行・カード |
| SCR-05 書籍詳細 | 書籍ヘッダー・メモ一覧 |
| SCR-06 全メモ検索 | メモ一覧 |

### アクション中のUI制御

- 保存・削除ボタンはアクション実行中に`disabled`にし、スピナーアイコンを表示する
- 二重送信を防ぐため、`useTransition` または Server Actions の `pending` 状態で制御する

## 10.2 エラーUI方針

### error.tsx（ページ単位）

```tsx
// 必ず "use client" が必要（Next.js仕様）
"use client"

export default function Error({
  error,
  reset,
}: {
  error: Error
  reset: () => void
}) {
  // エラーメッセージ + 再試行ボタンを表示
}
```

| 項目 | 内容 |
|---|---|
| 配置 | `(protected)/home/`・`books/`・`memos/` 配下 |
| 表示内容 | 「データの取得に失敗しました」＋再試行ボタン |
| 再試行 | `reset()` でError Boundaryをリセットし再レンダリング |

### トースト通知（アクション単位）

Server Actionsの結果に応じてトーストを表示する。

| ケース | トースト種別 | メッセージ例 |
|---|---|---|
| 書籍登録成功 | success | 「書籍を登録しました」 |
| 書籍削除成功 | success | 「書籍を削除しました」 |
| メモ保存成功 | success | 「メモを保存しました」 |
| バリデーションエラー | error | 「タイトルは必須です」 |
| DB登録失敗 | error | 「保存に失敗しました。もう一度お試しください」 |
| 認証エラー | error | 「ログインが必要です」→ログイン画面へリダイレクト |
| ネットワークエラー | error | 「通信エラーが発生しました。もう一度お試しください」 |

### 確認ダイアログ（削除操作）

| 対象 | 表示タイミング |
|---|---|
| 書籍削除 | MOD-02の削除ボタン押下時 |
| メモ削除 | MOD-04・SCR-08の削除ボタン押下時 |

メッセージ例：「この書籍と関連するすべてのメモを削除します。この操作は取り消せません。」
