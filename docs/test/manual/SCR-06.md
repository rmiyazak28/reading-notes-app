## 不合格時の記録欄

不合格（❌）となったケースは以下に記録する。

| # | 内容 | 対応状況 |
|---|---|---|
| 1-4 | ソートが登録日順/更新日順とあるにもかかわらず、登録日が表示されていない | 修正済み。１列に２行で登録日/更新日を表示するように変更。 |
| 2-3 | フィルタエリアの上部の余白が狭い（検索バーがクリックしずらい） | 修正済み。ヘッダーとの間に余白を確保。 |
| 3-1 | 検索欄に文字入力して検索したらエラー | 修正済み。サーバー側の OR フィルタを廃止。query がある場合でもサーバーは全件取得し、取得後のクライアント側フィルタ（既存コード）でメモ内容・書籍名・著者名・タグ名の横断絞り込みを行うよう統一。 |
| 3-2 | 書籍名で検索できない | 修正済み。query がある場合は range を外して全件取得し、クライアント側で絞り込む方式に変更。 |
| 7-1 | ★が即時反映されない | 修正済み。 星トグル専用の startFavoriteTransition / isFavoriteToggling を追加し、isPending（検索の pending 状態）とは独立して管理するよう分離。 |
| 9-2 | SCR-06全メモ検索画面に戻らない | 修正済み。router.push('/books/${book.id}') を router.back() に変更したため、SCR-06（/memos）から遷移した場合はSCR-06に、SCR-04/05（/books/[id]）から遷移した場合は書籍ページに、それぞれ戻るように変更。 |
| 11-4 | 先頭50件にリセットされない | 修正済み。クエリの場合はキャッシュして先頭50件のみ表示し、「もっと見る」はキャッシュから追加表示する方式に変更。 |
